Life cycle hooks

Lifecycle hooks are a special functionality in Angular that allow us to “hook into” and run code at a specific lifecycle event of a component or directive. Angular manages components and directives for us when it creates them, updates them, or destroys them. With lifecycle hooks, we can gain better control of our application. Components and directives both have life cycle hook, component have more life cycle hooks than directives.

Common lifecycle hooks in components and directives
Name Description
ngOnInit It will be invoked when a component is initialized. It call only once after the first ngOnChanges
ngOnChanges It is called once beforengOnInitand will be called whenever the data bound input property changes.
ngOnDestroy It will be called once just before the component is removed from DOM. In other words when the component is destroy from the dom. It is generally used to avoid memory leak.
ngDoCheck It invoked when the change detected of a given component is invoked. It is called immediately after ngOnChanges

Note: Constructor is not a life cycle hook. It is a typescript feature. Constructor calls first then ngOnInit calls.

Angular routing

It means navigation between pages. Navigation is one the important aspect in an application. SPA does not have multiple page concept it does move form one view to another view, so routing is an important concept in angular.

Name Description
router-outlet It is a routing directive and is used to inform angular that which component to be load when the path is matched.
routerLink Use this directive instead of href in a tag to avoid reloading in HTML pages. It is recommended in angular.
routerLinkActive It is a directive which is used to highlight active route or link.
Tips to improve performance of angular application
  1. Unsubscribe from Observables using ngOnDestroy which prevent memory leak.
  2. Make use of Lazy Loading to load only the necessary modules at the initial load.
  3. Using AoT Compilation.
  4. Using OnPush Change Detection Strategy (Change detection is the process by which angular updates the DOM.)
  5. Using Pure Pipes.
  6. Use trackBy option for For Loop.
  7. Avoid computation in template files.
  8. Usage of Web Workers.
Decorators

In angular decorators are the special kind of declarations and functions that allow a services, directives, filters to be modified before it is used. It is denoted by @ symbol at the beginning.

There are four main types of angular decorators:
  1. Class decorators, such as @Component and @NgModule
  2. Property decorators for properties inside classes, such as @Input and @Output
  3. Method decorators for methods inside classes, such as @HostListener
  4. Parameter decorators for parameters inside class constructors, such as @Inject
